# Page Performance Analysis Plan

## 1. Page Purpose

Describe what the page is meant to accomplish from both a business perspective and a user perspective.

## 2. Primary User Actions

List the most important actions users are expected to take on the page, separating intent actions from completed conversions.

## 3. Measurement Questions

List the key questions the analysis should answer about traffic, engagement, CTAs, content interaction, campaign performance, and user intent.

## 4. Recommended Metrics

List the metrics needed to evaluate the page against its purpose and expected user actions.

## 5. CTA and Event Tracking Requirements

Define which CTA clicks, events, scroll actions, form actions, outbound clicks, or flow starts need to exist or be validated before analysis.

## 6. Data Sources Needed

List the analytics, product behavior, campaign, vendor, and documentation sources needed for the analysis.

## 7. Segments and Breakdowns

List the useful breakdowns for the analysis, such as device, traffic source, campaign, offer category, audience, new vs returning users, or date range.

## 8. Data Caveats and Risks

Document likely reporting limitations, tracking gaps, attribution issues, third-party visibility gaps, consent impacts, or interpretation risks.

## 9. Analysis Output Structure

Recommend the structure of the final page performance report.

## 10. Next Agent to Run

Recommend the next agent or workflow that should run after the page performance plan is complete.
