# Data Source Map

## 1. Reporting Objective

Summarize the reporting question, initiative, page, campaign, or business decision this data map supports.

## 2. Required Data Sources

List each source needed for the report. For each source, include its purpose, what it can answer, and what it cannot answer.

## 3. Metric-to-Source Map

Map each required metric or metric category to its best source of truth. Separate delivery, traffic, engagement, CTA / intent, conversion, revenue, tracking validation, campaign taxonomy, and vendor outcome metrics where helpful.

## 4. Required Exports or Reports

List the specific reports, exports, screenshots, or summaries that need to be pulled from each source.

## 5. Required Fields

List the fields, dimensions, event parameters, or metrics that should be included in the data pulls.

## 6. Source Ownership and Access

Identify who likely owns or can provide each source. Clearly mark unknown ownership, missing access, or vendor dependencies.

## 7. Caveats by Source

Document source-specific limitations, data quality risks, interpretation caveats, platform mismatch risks, and vendor visibility gaps.

## 8. Manual Export Now vs Connect Later

Classify each source as Manual export now, Manual QA now, Connect later, Vendor request needed, or Not needed for this report. Explain the reason for each classification.

## 9. Data Readiness Assessment

Provide one readiness status and a short reason: Ready to Pull, Partially Ready, Not Ready — Missing Access, Not Ready — Missing Source Definition, or Not Ready — Vendor Dependency.

## 10. Next Agent to Run

Recommend the next agent or workflow after data source mapping.
